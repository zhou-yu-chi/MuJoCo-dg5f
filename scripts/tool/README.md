# 02_capture_to_mujoco.py 參數說明

RealSense 拍照 → 多視角融合 → 匯入 MuJoCo 的工具。物體放轉盤上，轉盤貼一張 ArUco marker，
繞著拍多張、用 marker 讀出每一幀的精確姿態，融合成封閉 mesh 再產生 `object.xml`。

執行一律用系統 `python3`（不要用 repo 的 `.venv`，那裡沒裝 open3d / pyrealsense2 / vhacdx）：

```bash
# 第一次用先印 marker
/usr/bin/python3 scripts/tool/02_capture_to_mujoco.py --make_marker marker.png

# 拍攝（p 拍一個視角、b 融合建置、c 清空重拍、q 離開）
/usr/bin/python3 scripts/tool/02_capture_to_mujoco.py --object_name my_cup --marker_length_m 0.037

# 不重拍，只用上次存的資料重新融合/建置（改重建參數時用這個）
/usr/bin/python3 scripts/tool/02_capture_to_mujoco.py --object_name my_cup --rebuild --tsdf_voxel_length 0.002
```

參數分兩類：**拍照階段**的參數會直接改動存到 `capture_debug/views.json` 的深度資料，改了一定要重拍才有效；
**重建階段**的參數只影響「融合/補洞/VHACD」這幾步，用 `--rebuild` 重跑就會套用新值，不用重新架相機。

---

## 拍照階段參數（改了要重拍才生效）

| 參數 | 預設值 | 作用 | 什麼情況調高 | 什麼情況調低 |
|---|---|---|---|---|
| `--marker_length_m` | 0.05 | marker 黑色方塊的實際邊長（公尺），決定整個模型的真實比例 | — | — |
| （沒有「調高調低」，用尺量印出來的 marker，量到多少填多少，量錯模型比例整個跑掉） |||||
| `--marker_max_reproj_err_px` | 3.0px | marker 姿態的重投影誤差門檻，超過視為這一幀偵測不可靠、直接拍攝失敗（見下方「已知限制」） | 常常拍不過、太嚴格時調大 | 融合出來的 mesh 出現一根根從中心射出去的放射狀尖刺（代表某幀姿態算錯了）時調小，逼它更嚴格地擋掉可疑的幀 |
| `--object_band_mm` | 60 | ROI 內判斷「這個深度算不算物體」的容忍帶寬（毫米） | 物體本身高低差大（例如立起來的娃娃、有明顯前後厚度），太窄會把物體遠端切掉一截時調高 | 轉盤/背景常常混進物體、去背抓太寬時調低（配合縮小 ROI 一起用） |
| `--depth_trunc` | 1.5m | 超過這個距離的深度資料整個丟棄 | 物體本身較大、或拍攝距離較遠時調高 | 背景雜訊多、想盡早排除遠處雜物時調低 |
| `--plate_z_clip_m` | 0.006m | 世界座標系（= marker/轉盤平面）高度低於這個值的像素，視為轉盤本身直接砍掉 | 融合出來的 mesh 還是常常把轉盤一起算進去時調高 | 物體貼著轉盤的底部細節（例如很矮的物體）被削掉太多時調低 |
| `--mask_erode_px` | 2px | 物體輪廓邊緣往內侵蝕的像素數，去掉深度相機在邊緣常見的「飛點」雜訊 | mesh 底部/邊緣出現一圈貼著地板往外攤開的薄殼、裙擺時調高 | 物體邊緣細節被吃掉太多、或視角數不夠多（<8）導致邊緣被侵蝕後出現破洞時調低 |
| `--min_views` | 6 | 至少要拍幾個視角才允許按 `b` 建置（純粹是最低門檻，不是建議值） | 想強制自己多拍一點再建置時調高 | 一般不用調 |
| `--marker_dict` / `--marker_id` | DICT_4X4_50 / 0 | ArUco 字典種類跟 marker 編號 | 要跟 `--make_marker` 印出來的那張一致，通常不用改 |||
| `--width` / `--height` / `--fps` | 640 / 480 / 30 | 相機串流解析度、幀率 | 一般不用調；電腦效能吃緊可以調低解析度/幀率 |||

## 重建階段參數（可以用 `--rebuild` 重跑，不用重拍）

| 參數 | 預設值 | 作用 | 什麼情況調高 | 什麼情況調低 |
|---|---|---|---|---|
| `--tsdf_voxel_length` | 0.003m（3mm） | TSDF 融合的體素大小，決定 mesh 最細能到多細 | 物體較大、想要表面更平滑、雜訊更少時調大（例如 0.005） | 物體較小（例如 <5cm）、想保留更多細節時調小（例如 0.0015） |
| `--tsdf_sdf_trunc` | 0.015m | TSDF 截斷距離，決定多少範圍內的深度量測會被平均在一起。經驗值大約是 `tsdf_voxel_length` 的 4~6 倍 | 多視角之間對齊有點誤差、表面顆粒感明顯時調大，換取更平滑但細節會糊一點 | 想要更貼近原始量測、保留細節時調小，但雜訊/顆粒感也會更明顯 |
| `--max_visual_triangles` | 40000 | 視覺 mesh 三角形數量上限，超過會自動簡化（decimation） | 想保留更多細節、不在意檔案變大/載入變慢時調高 | 想要模型輕量、載入快時調低 |
| `--max_hulls` | 32 | VHACD 最多分解成幾塊凸包（碰撞形狀） | 物體形狀複雜、凹陷多，碰撞形狀跟視覺差太多、容易穿模時調高 | 物理模擬變慢、或 MuJoCo 出現 solver 記憶體不足警告時調低 |
| `--max_neighbor_pose_jump_deg` | 60.0deg | 融合前的姿態異常檢查：跟拍攝順序上前後鄰居差超過這個角度、但跳過它前後卻能接得平順，就視為 marker 翻轉歧義造成的孤立壞幀，自動排除不參與融合 | 拍攝時故意每張都轉很大角度（例如每次轉 45 度以上）時調大，避免正常的大角度被誤判 | 一般不用調 |

## 其他（跟建模品質無關，只影響顯示）

| 參數 | 說明 |
|---|---|
| `--no_view` | 建置完不要自動開 MuJoCo viewer 預覽 |
| `--preview_with_hand` | 預覽視窗連手一起顯示（預設只顯示地板 + 物件，不含手，看起來比較乾淨） |

## 已知限制

- 物體貼著轉盤的底面從頭到尾不會被拍到，這是單一轉盤掃描的物理限制，靠 `cap_mesh_holes()` 自動補一塊平面蓋住洞口，不是真正掃描到的形狀。
- 視覺 mesh 表面凹凸不平（深度感測器雜訊 + 多視角對齊誤差）幾乎不影響抓取——MuJoCo 實際拿來算碰撞的是 VHACD 分解出的凸包，不是這個視覺 mesh。
- marker 如果被物體或手擋住那一幀會直接拍攝失敗，相機盡量架高、用較大俯角往下拍，marker 放在轉盤上離物體遠一點的位置。
- 平面正方形 marker 在某些角度下 solvePnP 有「翻轉歧義」——正確姿態跟鏡射過去的姿態（往往剛好差 180 度）重投影誤差可能非常接近，連 `SOLVEPNP_IPPE_SQUARE` 挑最小誤差都可能挑錯，實測抓過一次真實案例：27 個視角裡有 1 個被誤判翻轉，融合出來整個變形。單靠重投影誤差沒辦法保證擋掉這種情況，所以另外加了 `reject_pose_outliers()`：拍攝順序上姿態該平滑漸變，跟前後鄰居都差很多、但跳過它前後卻能接得平順的那一幀，判定是孤立壞幀直接排除（見 `--max_neighbor_pose_jump_deg`）。如果又看到 mesh 變形、放射狀尖刺、或缺一大塊，先試 `--rebuild`（會印出排除了哪些視角），不用重拍；如果被排除的視角太多導致覆蓋不夠，才需要針對那幾個角度重拍。
